# Code Mode Delivery

- Refuse to start implementation unless there is a linked issue or an explicit instruction to create one immediately afterward.
- Work on a feature branch, not `main`.
- Prefer TDD: Write the smallest failing test first, make it pass, then refactor.
- Maintain the test pyramid: Unit (logic), Integration (boundaries/services), E2E (workflows).
- (Optional) Before implementing, use Codeix for symbol discovery if paths are unclear.
- Update the issue change record and relevant tests whenever behavior changes.
